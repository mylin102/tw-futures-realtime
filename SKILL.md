# Skill: Squeeze Taiwan Futures Logic (Ultimate)

此文件定義了 `squeeze-tw-futures-realtime` 專案的高階分批操作邏輯。

## 1. 資金與部位管理 (Position Sizing)
*   **預設規模**: 每次進場固定 **2 口** (TMF/MX)。
*   **資金檢查**: 系統自動檢核可用保證金是否足以支付目標口數。

## 2. 分批停利策略 (Partial Exit Logic)
為了平衡獲利穩定性與趨勢追蹤，系統採用專業的二階段出場：
1.  **第一階段停利 (TP1)**: 當帳面獲利達到 **+40 點** 時，立即平倉 **1 口**。
    *   *目的：確保獲利入袋，抵銷手續費與磨損。*
2.  **第二階段奔跑 (Runner)**: 剩餘的 **1 口** 繼續持有。
    *   *保護機制：一旦 TP1 達成，Runner 的停損點自動移至「進場成本價」。*
    *   *最終出場：直到發生「趨勢轉弱 (Score < 20)」或「反向強勢訊號」才全數出清。*

## 3. 環境與進場過濾
*   **中線過濾**: 15m EMA 60 決定多空主場。
*   **開盤強弱 (Opening Regime)**: 每日開盤價關係判定當日主場趨勢。
*   **進場閾值**: MTF 分數絕對值 >= 70。

## 4. 風險控管
*   **初始停損**: 進場後設定 **30 點** 硬性停損 (2 口同步)。
*   **保本停損**: TP1 後自動執行。
*   **結構停損**: 穿透 VWAP 線時出場 (若與當日開盤強弱不符)。
*   **留倉設定**: 允許留倉過夜 (Swing Mode)，以捕捉完整跨日波段。
